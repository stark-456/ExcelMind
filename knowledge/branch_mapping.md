# 分公司名称映射规则

## 规则描述

用户查询时可能使用简称或别名，需要正确映射到数据表中的标准名称。

## 常见映射

| 用户输入 | 实际字段值 |
|----------|-----------|
| 东城 | 东城分公司 |
| 西城 | 西城分公司 |
| 朝阳 | 朝阳分公司 |
| 海淀 | 海淀分公司 |

## 注意事项

1. **模糊匹配优先**：如果用户输入"东城"，应使用 `contains` 匹配包含"东城"的值
2. **不要假设后缀**：不同表可能有不同的命名规范（如"东城"、"东城分公司"、"东城区"）
3. **先查唯一值**：如果不确定具体值，先使用 `get_unique_values` 查看该列的实际值

## 正确示例

当用户问"东城的数据"时，建议先查询或使用 contains：
```json
{"tool": "filter_data", "args": {"column": "分公司名称", "operator": "contains", "value": "东城"}}
```
